## 审查框架

### 一、大纲基本信息审查

| 审查项目 | 审查要点 | 审查结果 |
|----------|----------|----------|
| 课程名称 | 是否规范、与培养方案一致 | |
| 课程代码 | 是否完整、编码规范 | |
| 学分/学时 | 总学时与学分是否匹配，理论与实践比例是否合理 | |
| 课程性质 | 必修/选修标注是否明确 | |
| 先修课程 | 是否列出，与课程体系的衔接是否合理 | |
| 开课单位 | 信息是否完整 | |
| 编制人/审核人 | 签名和日期是否齐全 | |

### 二、课程目标审查

1. **目标表述规范性**
   - 是否使用可观测、可测量的行为动词
   - 是否避免使用"了解""掌握"等模糊动词
   - 目标数量是否适当（一般3-5条）

2. **目标与培养方案的对应关系**
   - 每条课程目标是否对应毕业要求指标点
   - 对应关系是否合理、可支撑
   - 是否形成完整的支撑矩阵

3. **目标层次合理性**
   - 是否覆盖不同认知层次（知识、能力、素养）
   - 目标之间是否存在逻辑递进关系
   - 是否体现高阶思维能力的培养

### 三、教学内容审查

1. **内容体系完整性**
   - 各章节/模块是否有明确的教学内容描述
   - 重难点是否标注清晰
   - 学时分配是否合理

2. **内容与目标的一致性**
   - 教学内容是否全面支撑课程目标
   - 是否存在冗余内容或缺失内容
   - 知识点的深度和广度是否适当

3. **内容的时代性与前沿性**
   - 是否融入学科最新发展成果
   - 是否反映行业最新标准和规范
   - 案例和素材是否具有时效性

### 四、教学方法审查

- 教学方法描述是否具体（而非简单罗列方法名称）
- 教学方法与教学内容的匹配性
- 是否体现以学生为中心的教学理念
- 是否包含信息化教学手段的应用
- 实践教学环节的设计是否充分

### 五、考核方式审查

1. **考核方案设计**
   - 考核方式是否多元（过程性考核 + 终结性考核）
   - 各考核环节的权重分配是否合理
   - 考核频次是否适当

2. **考核与目标的对应关系**
   - 每项考核是否对应具体的课程目标
   - 考核方式是否能有效检测目标达成度
   - 评分标准是否明确、可操作

3. **考核标准明确性**
   - 各等级评分标准是否清晰
   - 过程性考核的评分细则是否完整
   - 是否有成绩评定方法说明

| 考核项目 | 权重 | 对应课程目标 | 评分标准明确性 |
|----------|------|-------------|---------------|
| 期末考试 | | | |
| 平时作业 | | | |
| 课堂表现 | | | |
| 实验/实训 | | | |

### 六、教材与参考资源审查

- 教材选用是否恰当（权威性、时效性、适用性）
- 参考书目是否充足且标注规范
- 是否推荐网络学习资源
- 教材与教学内容的匹配程度

### 七、课程思政审查

- 是否明确课程思政目标
- 思政元素与专业内容是否有机融合
- 是否避免生硬的"贴标签"式融入
- 思政融入点是否具体可行

### 八、格式规范审查

- 目录结构是否完整
- 排版格式是否统一规范
- 术语使用是否一致
- 是否有明显的文字错误

### 第二部分：详细审查意见

对每个维度逐一给出：
- **亮点**：做得好的方面（如适用）
- **问题**：具体指出不规范或缺失之处
- **建议**：改进的具体方案（附修改建议的示例文字）

### 第四部分：修订建议总结

- 必须修改的事项（影响大纲合规性）

- 可以完善的事项（锦上添花的改进）

## 审查原则

- 依据国家标准和学校规范进行审查，不凭主观印象
- 指出问题的同时给出具体可操作的修改建议
- 尊重课程特色，不强求所有课程用同一模板
- 关注实质内容而非仅关注格式
- 区分"规范性问题"和"优化建议"，前者必须修改
